# WordHub 产品需求文档

当前版本：V0.4（后端接入就绪版）
更新日期：2026-03-12

## 1. 文档定位

本文档描述的是 WordHub 当前的产品形态与交互逻辑，目标是让产品、设计、前端在同一个事实上协作。

本文档聚焦于用户侧的功能体验与业务流程，尽量屏蔽底层技术实现细节。

## 2. 产品概述

WordHub 是一个以“视觉化记忆单词”为核心的轻量学习应用。产品核心流程包含 4 个连续步骤：

1. 用户输入英文单词。
2. 系统通过 AI 引擎（或本地预设库）生成 3:4 记忆卡片数据。
3. 用户在结果页浏览生成的可视化结果。
4. 用户进入复习页，通过图片线索完成 4 选 1 记忆训练。

当前版本采用移动优先设计，并兼容 Web 宽屏体验。

## 3. 当前版本范围

### 3.1 已实现范围

- 查询页：输入单词、填充示例词库、清空输入、发起生成任务
- 结果页：3:4 结果卡片浏览、左右切换、返回、进入复习
- 复习页：仅保留图片模式，4 选 1 判题、反馈动画、自动切题
- 个人页：登录/注册流程、BYOK 双引擎配置、退出登录
- **用户信息管理**：支持头像展示与更新
- 全局底部导航：查询 / 复习 / 我的

### 3.2 未实现范围

- 真实的支付与订阅系统
- 跨设备的实时云端同步（部分数据仍依赖本地缓存）
- 社交分享功能

## 4. 信息架构

当前应用包含 4 个一级页面：

- **查询页**：核心入口，承载输入与生成。
- **生成结果页**：中间态，展示生成内容。
- **单词复习页**：沉浸式学习场景。
- **个人信息页**：账号与配置中心。

其中：

- 查询页、复习页、个人页展示底部导航栏。
- 结果页隐藏底部导航栏，专注于内容浏览与下一步操作。

## 5. 核心用户流程

### 5.1 查询到结果

1. 用户在查询页输入一个或多个英文单词。
2. 系统对输入内容进行标准化处理（去重、格式化）。
3. 点击“AI 智能生成”后，系统发起生成任务。
4. 任务完成后，生成一组视觉记忆卡片。
5. 页面自动跳转到结果页。

### 5.2 结果到复习

1. 用户在结果页左右切换结果卡片。
2. 点击底部“开始复习”按钮。
3. 页面跳转到复习页。
4. 复习页直接使用最近一次生成结果作为当前复习卡组。

### 5.3 未登录到已登录

1. 用户进入个人页时，默认是未登录状态。
2. 点击顶部个人卡片，唤起登录/注册弹窗。
3. 用户输入账号信息完成认证。
4. 登录成功后，页面展示用户信息，并解锁高级配置功能。

## 6. 页面需求

## 6.1 查询页

### 页面目标

让用户快速输入待学习单词，并生成一组可复习的视觉记忆卡片。

### 当前 UI 结构

- 顶部居中 WordHub Logo
- 中部白色输入卡片
- 卡片顶部一行辅助说明
- 右上角两个幽灵按钮：
  - 填充示例词库
  - 清空
- 中部大文本输入框
- 底部品牌渐变主按钮“AI 智能生成”
- 页面底部悬浮 Tab Bar

### 交互逻辑

- 输入支持逗号、空格、换行混合分隔。
- 输入内容会被自动标准化，并限制单词数量（最多 12 个）。
- 失焦时自动格式化。
- “填充示例词库”会填入默认示例：
  - `vivid, anchor, serene, pulse, orchard, glacier, agile, bloom, lantern, horizon, prism, forge`
- “清空”按钮清空当前输入。
- 点击“AI 智能生成”后：
  - 校验单词合法性。
  - 若已登录并连接后端，发起远程生成任务；否则使用本地库生成。
  - 加载成功后跳转至结果页。

## 6.2 生成结果页

### 页面目标

让用户查看本次生成的单词记忆卡片，并决定是否进入复习。

### 当前 UI 结构

- 顶部 Header
  - 左侧返回箭头
  - 中间 WordHub Logo
  - 右侧进度胶囊，如 `1 / 6`
- 中部结果浏览区
  - 桌面端显示左右切换箭头
  - 中央为单张 3:4 结果卡片
- 结果卡片结构
  - 顶部英文单词
  - 中部 3:4 图片区域
  - 右下角下载按钮
  - 底部中文释义与视觉提示文案
- 底部固定 CTA：“开始复习”

### 交互逻辑

- 支持手势滑动、键盘方向键或点击箭头切换卡片。
- 顶部进度胶囊实时显示当前索引
- 返回按钮：
  - 有历史页面时返回上一级
  - 无历史页面时回到查询页
- 点击“开始复习”后跳转到复习页
- 点击下载按钮时，提供图片保存反馈。

## 6.3 单词复习页

### 页面目标

让用户围绕“图像线索 + 四选一”完成快速记忆训练。

### 当前产品定位

- 已彻底废弃“文字模式”
- 当前只保留“图片模式”

### 当前 UI 结构

- 顶部 Header
  - 左侧 WordHub Logo
  - 右侧进度胶囊，如 `1 / 6`
- 中部主视觉卡片
  - 固定为 3:4 竖版比例
  - 使用整张图片区域铺满
  - 底部有黑色渐变遮罩
  - 遮罩上显示英文单词与中文视觉提示
- 下方四个中文选项按钮
- 页面底部悬浮 Tab Bar

### 判题状态

每道题存在 3 种状态：未作答、答对、答错。

### 判题逻辑

- 用户点击任意选项后，立即锁定所有选项
- 锁定期间不允许重复点击或切题

### 答对反馈

- 正确选项变绿、高亮。
- 其他 3 项降透明并灰阶处理
- 800ms 后自动进入下一题

### 答错反馈

- 错误选项变红并抖动。
- 正确选项强制变绿提示。
- 剩余 2 项降透明
- 1500ms 后自动进入下一题

### 切题逻辑

- 自动切题前会展示反馈动画
- 切题时重置：
  - 当前选中项
  - 答题状态
- 卡片支持左右切换动效

## 6.4 个人信息页

### 页面目标

承载登录入口、AI 配置入口与退出登录操作。

### 当前默认状态

应用首次进入时默认未登录。

### 未登录态 UI

- 顶部个人卡片仅显示：
  - 默认灰色头像
  - `点击登录 / 注册`
  - 一句简短提示文案
- 中部 BYOK 配置卡片降透明
- BYOK 卡片上覆盖半透明锁定层与锁图标
- 锁定文案为：
  - `登录后配置专属 AI 大脑`
- 底部不显示退出登录按钮

### 已登录态 UI

- 顶部个人卡片显示：
  - 头像 `LM`
  - 用户名 `Luna Moss`
  - **支持点击头像上传/更新个人图片**
- 中部显示完整 BYOK 配置卡片
- 底部显示白底红字“退出登录”按钮

### 顶部个人卡片交互

- 未登录时点击个人卡片：打开登录弹窗
- 已登录时点击头像：**唤起图片选择器，进行头像更新**

## 6.5 登录弹窗

### 页面目标

以最轻的方式完成演示级登录，并兼容 H5 端 Autofill。

### 弹窗结构

- 居中白色弹窗
- 顶部右上角关闭按钮
- 中部 Logo
- 提示文案：
  - `欢迎回到视觉词汇宇宙`
- 两个 Tab：
  - 密码登录
  - 验证码登录

### 密码登录

- 使用 `<form>` 包裹
- 邮箱输入框
  - H5：`type="email"`，带 `autocomplete="email"`
- 密码输入框
  - H5：`type="password"`，带 `autocomplete="current-password"`
- 右下角有“忘记密码？”文本链接
- 底部主按钮：
  - `登录 / 注册`

### 验证码登录

- 使用 `<form>` 包裹
- 邮箱输入框
- 验证码输入框
  - H5：带 `autocomplete="one-time-code"`
- 右侧“获取验证码”按钮
- 底部主按钮：
  - `登录 / 注册`

### 当前交互逻辑

- 点击提交后进行表单校验。
- 发起网络请求进行认证（或注册）。
- 认证成功后，关闭弹窗，刷新个人页状态。

## 6.6 BYOK 大模型双擎配置

### 页面目标

允许用户在个人页手动保存自己的模型服务商与 API Key。

### 当前结构

卡片标题：

- `大模型双擎配置 (BYOK)`

包含两段配置：

1. `🧠 文字与翻译大脑`
2. `🎨 视觉渲染引擎`

中间以细分隔线分开。

### 文字与翻译大脑

支持 4 个服务商：

- DeepSeek
- 千问
- OpenAI
- Gemini

字段包含：

- 服务商胶囊切换
- API Key 密码输入框
- 当前接口域名提示
- “前往官方申请 Key”链接

### 视觉渲染引擎

支持 3 个服务商：

- OpenAI
- 千问
- Gemini

明确不包含：

- DeepSeek

字段包含：

- 服务商胶囊切换
- API Key 密码输入框
- 当前接口域名提示
- “前往官方申请 Key”链接

### 服务商映射字典

- DeepSeek
  - Base URL: `https://api.deepseek.com`
  - Key Link: `https://platform.deepseek.com/`
- 千问
  - Base URL: `https://dashscope.aliyuncs.com/compatible-mode/v1`
  - Key Link: `https://help.aliyun.com/zh/model-studio/get-api-key`
- OpenAI
  - Base URL: `https://api.openai.com/v1`
  - Key Link: `https://platform.openai.com/api-keys`
- Gemini
  - Base URL: `https://generativelanguage.googleapis.com`
  - Key Link: `https://aistudio.google.com/app/apikey`

### 联动逻辑

若用户在“文字与翻译大脑”中选择以下服务商之一，并输入 Key：

- OpenAI
- 千问
- Gemini

则系统会在 `imageLinked = true` 的前提下自动同步：

- 绘图服务商
- 绘图 API Key

### 解绑逻辑

用户一旦手动修改以下任一项，就会触发解绑判断：

- 绘图服务商
- 绘图 API Key

若当前绘图配置不再与文字配置一致，则 `imageLinked` 变为 `false`。

### 保存逻辑

- 点击“保存配置”后，将配置写入 `wordhub:ai-config`
- 保存成功后弹出 Toast
- 当前版本仅做本地持久化，不会立即驱动生成链路

## 7. 底部导航栏

### 当前范围

底部导航仅出现在：

- 查询页
- 复习页
- 个人页

### 当前样式

- 单层悬浮毛玻璃胶囊
- 固定在页面底部上方
- 三个等宽按钮：
  - 查询
  - 复习
  - 我的
- 选中态只改变图标与文字颜色，不加额外色块
- 深色背景场景下已加强底色、边框与阴影对比度

### 当前交互逻辑

- 点击当前 Tab：无动作
- 点击其他 Tab：`reLaunch` 到目标页面

### 防遮挡策略

查询页、复习页、个人页的主内容容器都补了额外底部留白，确保最后一张卡片或最后一个按钮不会被悬浮导航挡住。

## 8. 本地数据与状态管理

## 8.1 Storage Key

当前使用的本地存储键如下：

- `wordhub:bootstrapped`
- `wordhub:auth-version`
- `wordhub:session`
- `wordhub:user`
- `wordhub:stats`
- `wordhub:results`
- `wordhub:ai-config`

## 8.2 登录态

- `wordhub:session = '1'`：已登录
- `wordhub:session = '0'`：未登录

当前版本通过本地 session 模拟登录态。

## 8.3 默认用户

已登录默认用户为：

- 用户名：`Luna Moss`
- 邮箱：`luna@wordhub.ai`
- 头像：`LM`
- 学习计划：`CET-6 Visual Sprint`
- 连续打卡：`12`

说明：

- 邮箱与学习计划当前主要保留在本地数据层
- 个人页 UI 目前只展示头像与用户名

## 8.4 复习统计

统计字段包括：

- `reviewed`
- `mastered`
- `accuracy`
- `streak`
- `todayGoal`
- `todayDone`
- `totalSessions`
- `librarySize`
- `correctCount`

### 登录前

- 未登录用户读取到的是空统计结构
- 不做真实累计

### 登录后

- 登录成功会写入默认统计
- 每次答题会更新：
  - `reviewed + 1`
  - `correctCount + 1`（仅答对）
  - `mastered + 1`（仅答对）
  - `totalSessions + 1`
  - `todayDone + 1`
  - `accuracy` 重新计算

### 生成结果时的额外更新

登录状态下生成结果后：

- `librarySize` 至少更新为 `results.length * 4`
- `todayDone` 增加 `min(results.length, 4)`

## 8.5 AI 配置结构

`wordhub:ai-config` 当前结构：

- `textProvider`
- `textApiKey`
- `imageProvider`
- `imageApiKey`
- `imageLinked`

默认值：

- `textProvider = deepseek`
- `textApiKey = ''`
- `imageProvider = openai`
- `imageApiKey = ''`
- `imageLinked = true`

## 8.6 结果数据结构

每个结果项当前包含以下核心字段：

- `id`
- `word`
- `meaning`
- `note`
- `scene`
- `visualCueZh`
- `example`
- `imagePrompt`
- `promptSections`
- `cameraStyle`
- `posterBackground`
- `accentColor`
- `promptVersion`
- `choices`

其中：

- `choices` 直接供复习页四选一使用
- `posterBackground` 当前承担图片区视觉氛围，但不代表真实图片资源

## 9. 响应式与端侧说明

### 9.1 移动优先

当前所有页面均以手机视觉为主设计。

### 9.2 Web 宽屏

H5 宽屏下：

- 页面会进入一个居中的“桌面画布”容器
- 主要内容不会无限拉宽
- 底部导航保持悬浮固定
- 查询页、复习页、个人页与底部导航在桌面端宽度关系已做统一优化

### 9.3 微信小程序

当前交互已支持构建到 `mp-weixin`，但部分 H5 专属能力会降级：

- `window.open` 会退化为复制链接
- 浏览器 Autofill 只在 H5 端完整生效

## 10. 当前限制

以下能力当前仍属于演示或占位状态：

- 查询页的“AI 智能生成”并未调用用户填写的 BYOK 配置
- 结果页图片区仍为占位图层，不是真实图片资源
- 下载按钮仅提示，不执行文件保存
- 验证码登录不发送真实验证码
- 登录 / 注册不接服务端账号体系
- 复习统计仅在本地维护，不支持云同步

## 11. 当前验收口径

当前版本可视为“高保真前端原型 + 本地状态机验证版”，验收标准如下：

### 查询页

- 能输入、格式化、去重单词
- 能填充示例词库
- 能完成模拟生成并跳转结果页

### 结果页

- 能查看 3:4 卡片
- 能左右切换
- 顶部 Header 为：左返回、中 Logo、右进度
- 能进入复习页

### 复习页

- 仅有图片模式
- 4 个选项可判题
- 答对 800ms 自动切题
- 答错 1500ms 自动切题
- 切题前后状态能正确重置

### 个人页

- 默认可看到未登录态
- 可打开登录弹窗
- 可模拟登录成功
- 登录后可配置并保存 BYOK
- 可退出登录并回到未登录态

## 12. 下一阶段建议

下一阶段建议按以下顺序推进：

1. 接入真实图片生成服务，打通 BYOK 到生成链路
2. 将结果页图片占位替换为真实图片资源
3. 将登录流程替换为真实账号体系
4. 引入云端结果与统计同步
5. 增加复习历史、错题沉淀与掌握度分层
